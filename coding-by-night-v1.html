Moderne web

Her er et kort overblik over nogle udvalgte aspekter af moderne webudvikling:

Semantisk HTML
Brugen af semantiske tags (fx header, main & footer) frem for generelle tags (fx div).
Accessibility (a11y)
Design for mange typer mennesker med fokus på tilgængelighed, fx ARIA-attributter og semantiske tags, som gør det lettere for skærmlæsere.
Responsivt design (mobile-first)
Design af websider, som egner sig til alle størrelser skærme. Gøres fx ved hjælp af relative størrelser (fx em eller vh) frem for absolutte (fx px) eller brugen af CSS media queries til at variere styling afhængig af fx skærmstørrelse.
Progressive enhancement
Fokus på at designe en simpel base (HTML), som virker for alle, og så kan man tilføje mere avancerede features (CSS og JavaScript) "ovenpå".

CSS-frameworks

CSS-frameworks er en slags "færdigbagt" CSS, som kan integreres ind i et projekt. Typisk fungerer de ved at opsætte et basalt layout samt definere en række CSS-klasser, som kan bruges overalt.

Typisk bruges CSS-frameworks til følgende:

  • Responsiv CSS gennem brug af klasser. Denne eksempelkode (Bootstrap) sætter, at der skal være en venstre margen på store skærme (laptop), men ingen på små (mobil): <main class="mb-0 mb-lg-5">. Det kunne man også selv gøre vha. CSS media queries, men frameworket har på forhånd defineret klasser til det.
  • Grid-baseret kolonnesystem, hvor man kan definere bredden af elementer som en ratio, fx 1/5 eller 3/12 af den mulige bredde. Gør det nemt at lave flot layout.

Denne side bruger Pure.css til at definere styling for tabellen ovenfor. Prøv selv at kigge i kildekoden.

Udvalgte CSS-frameworks

  • Pure.css: Et minimalt framework, som kun fylder 3,5 kb
  • Bootstrap: Et noget mere omfattende framework, som kan en del mere.
